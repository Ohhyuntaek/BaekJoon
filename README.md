# 📚 코딩 테스트 자료구조 & 알고리즘 정리

## 🧱 1. 자료구조별 정리

### 🔹 배열 (Array)

- **주요 알고리즘**: 정렬, 투 포인터, 슬라이딩 윈도우
- **활용 예시**: 연속 부분 합, 정렬 후 처리
- **대표 문제**: K번째 수, 두 수의 합 (Two Sum)

### 🔹 링크드 리스트 (Linked List)

- **정의**: 노드들이 포인터(또는 참조)로 연결된 선형 자료구조
- **특징**:
    - 동적 메모리 할당 → 크기 조절이 유연함
    - 삽입/삭제가 빠름 (중간 위치 기준 O(1))
    - 인덱스 접근은 느림 (O(n)), 순차 접근만 가능
- **활용 예시**:
    - 해시 충돌 처리 (Chaining)
    - 스택/큐 구현
    - 캐시 구조 (LRU 등)
- **연결 리스트 종류 비교**
    
    
    | 유형 | 포인터 구조 | 장점 | 단점 | 대표 활용 |
    | --- | --- | --- | --- | --- |
    | 단일 연결 리스트 (Singly Linked List) | `next`만 가짐 | 구조 단순, 메모리 절약 | 역방향 탐색 불가 | 스택, 기본 연결 리스트 |
    | 이중 연결 리스트 (Doubly Linked List) | `prev`, `next` 모두 존재 | 양방향 탐색 가능, 중간 삭제 용이 | 메모리 사용 증가, 구현 복잡 | LRU 캐시, 텍스트 편집기 |
    | 원형 연결 리스트 (Circular Linked List) | 마지막 노드의 `next`가 처음 노드 가리킴 | 순환 구조 처리 용이 | 탐색 종료 조건 주의 필요 | 원형 큐, 조세퍼스 문제 |

### 🔹 스택 (Stack)

- **특징**: 후입선출 (LIFO)
- **활용 예시**: 괄호 유효성 검사, 후위 표기법 계산
- **대표 문제**: 올바른 괄호, 탑

### 🔹 큐 (Queue)

- **특징**: 선입선출 (FIFO)
- **활용 예시**: BFS, 캐시, 프로세스 관리
- **대표 문제**: 프린터, 다리를 지나는 트럭

### 🔹 덱 (Deque)

- **특징**: 양방향 입출력 가능
- **활용 예시**: 슬라이딩 윈도우, 회전 큐
- **대표 문제**: 회전하는 큐, 슬라이딩 윈도우 최대값

### 🔹 힙 (Heap)

- **정의**: 완전 이진 트리를 기반으로 한 우선순위 큐 구현용 자료구조
- **종류**:
    - 최소 힙 (Min Heap): 부모 노드 ≤ 자식 노드
    - 최대 힙 (Max Heap): 부모 노드 ≥ 자식 노드
- **특징**:
    - 삽입, 삭제: O(log n)
    - 루트 노드가 항상 최댓값 또는 최솟값
- **활용 예시**:
    - 우선순위 큐, 다익스트라 알고리즘
    - 힙 정렬, 작업 스케줄링

### 🔹 그래프 (Graph)

- **정의**: 정점(Vertex)과 간선(Edge)으로 구성된 비선형 자료구조
- **종류**:
    - 방향 그래프 / 무방향 그래프
    - 가중치 그래프 / 비가중치 그래프
- **표현 방법**:
    - 인접 행렬 (2D 배열)
    - 인접 리스트 (딕셔너리 + 리스트)
- **활용 예시**:
    - 경로 탐색 (DFS/BFS)
    - 최단 경로 (다익스트라 등)
    - 사이클 판별, 위상 정렬

### 🔹 트리 (Tree)

- **정의**: 사이클이 없는 연결 그래프의 특수한 형태
- **특징**:
    - 계층 구조
    - N개의 노드 → N-1개의 간선
- **종류**:
    - 이진 트리, 이진 탐색 트리 (BST)
    - 완전 이진 트리, 균형 트리 (AVL, Red-Black)
- **활용 예시**:
    - 이진 탐색, 표현식 계산
    - LCA(최소 공통 조상), 트리 DP

### 🔹 우선순위 큐 (Priority Queue)

- **특징**: 높은 우선순위를 가진 요소가 먼저 나옴 (최소/최대 힙 기반)
- **활용 예시**: 다익스트라 알고리즘, 작업 스케줄링
- **대표 문제**: 이중 우선순위 큐, 카드 정렬하기, 네트워크 연결

### 🔹 해시 (Hash Table)

- **활용 예시**: 빠른 검색, 중복 제거, 빈도 계산
- **대표 문제**: 완주하지 못한 선수, 전화번호 목록

### 🔹 페어 (Pair)

- **정의**: 두 개의 데이터를 하나로 묶는 구조 (Python에서는 `tuple`로 표현)
- **언어별 표현**:
    - Python: `(a, b)`
    - C++: `pair<int, int>`
- **활용 예시**:
    - 좌표 표현: `(x, y)`
    - 그래프 간선 정보: `(노드, 가중치)`
    - 우선순위 큐 요소: `(우선순위, 값)`
- **대표 문제**:
    - 다익스트라 (힙에 `(거리, 노드)` 저장)
    - BFS/DFS에서 `(x, y)` 위치 저장

### 🔹 튜플 (Tuple)

- **정의**: 여러 개의 값을 하나의 고정된 순서로 묶은 자료구조 (수정 불가, immutable)
- **특징**:
    - 불변 (immutable)
    - 해시 가능 → 딕셔너리/집합의 키로 사용 가능
- **활용 예시**:
    - 2차원 이상의 고정된 좌표 저장: `(x, y, z)`
    - 상태 표현 (예: DP 메모이제이션 키)
- **대표 문제**:
    - 방문한 상태 기록 (예: visited set에 튜플 저장)
    - DP with memoization (key로 튜플 사용)

### 🔹 딕셔너리 (Dictionary, Hash Map)

- **정의**: Key-Value 쌍으로 데이터를 저장하는 해시 기반 자료구조
- **특징**:
    - 평균 시간복잡도 O(1) (검색, 삽입, 삭제)
    - 키는 해시 가능해야 함 (숫자, 문자열, 튜플 등)
- **활용 예시**:
    - 빈도 수 저장
    - 메모이제이션
    - 그래프 인접 리스트
- **대표 문제**:
    - 완주하지 못한 선수
    - 전화번호 목록
    - Two Sum

### 🔹 비트마스크 (Bitmask)

- **정의**: 정수의 이진 표현을 사용해 집합, 상태 등을 효율적으로 표현
- **특징**:
    - 집합의 포함 여부, 상태 전이 등 O(1) 연산
    - 조합/부분 집합 생성에 유리
- **활용 예시**:
    - 상태 압축 DP
    - 부분 집합 탐색
- **대표 문제**:
    - 외판원 순회 (백준 2098)
    - 집합 (백준 11723)

### 🔹 트라이 (Trie)

- **정의**: 문자열을 문자 단위로 트리 형태로 저장하는 구조
- **특징**:
    - 검색, 삽입: O(L) (L = 문자열 길이)
    - 메모리 사용 많음 (노드 분기 많음)
- **활용 예시**:
    - 자동완성, 접두사 검색
    - 문자열 필터링 (금지어 등)
- **대표 문제**:
    - 전화번호 목록 (프로그래머스)
    - 가사 검색 (프로그래머스)

### 🔹 유니온 파인드 (Disjoint Set)

- **정의**: 여러 집합을 하나의 자료구조로 표현하고, 합치기와 같은 집합 판별에 사용
- **특징**:
    - Union, Find 연산: 평균 O(α(n)) (거의 상수)
    - 경로 압축 최적화 기법 활용
- **활용 예시**:
    - 네트워크 연결 여부 확인
    - 크루스칼 알고리즘
- **대표 문제**:
    - 집합의 표현 (백준 1717)
    - 여행 가자 (백준 1976)

### 🔹 세그먼트 트리 (Segment Tree)

- **정의**: 구간 쿼리(합, 최소, 최대 등)를 빠르게 처리하기 위한 트리 구조
- **특징**:
    - 구간 쿼리 및 업데이트: O(log n)
    - 배열을 트리로 재구성
- **활용 예시**:
    - 구간 합/최솟값/최댓값 쿼리
    - 동적 수열 문제
- **대표 문제**:
    - 구간 합 구하기 (백준 2042)
    - 수열과 쿼리

### 🔹 펜윅 트리 (Fenwick Tree, Binary Indexed Tree)

- **정의**: 누적합(부분합)을 빠르게 처리하는 트리 구조
- **특징**:
    - 구현이 간단한 세그먼트 트리
    - 업데이트/쿼리 O(log n)
- **활용 예시**:
    - 역순 쌍 개수 계산
    - 순위 통계
- **대표 문제**:
    - 수들의 합 4 (백준 11659)
    - 역전 쌍 (Leetcode)

## ⚙️ 2. 알고리즘별 정리

### 🔸 정렬 (Sorting)

- **정의**: 데이터를 순서에 따라 정렬하는 알고리즘
- **대표 알고리즘**:
    
    
    | 정렬 알고리즘 | 시간 복잡도 | 안정성 | 주요 특징 |
    | --- | --- | --- | --- |
    | 버블 정렬 | O(n²) | 안정정렬 | 인접 요소를 반복 교환 |
    | 선택 정렬 | O(n²) | 불안정정렬 | 최소값을 선택하여 앞에 배치 |
    | 삽입 정렬 | O(n²) | 안정정렬 | 정렬된 구간에 원소 삽입 |
    | 병합 정렬 | O(n log n) | 안정정렬 | 분할 정복 기반, 병합 과정 필요 |
    | 퀵 정렬 | 평균 O(n log n) / 최악 O(n²) | 불안정정렬 | 피벗 기준 분할, 빠르지만 불안정 |
    | 힙 정렬 | O(n log n) | 불안정정렬 | 힙 자료구조 사용, 선택 정렬 기반 |
    | 계수 정렬 | O(n + k) | 안정정렬 | 정수 데이터에 한정, 매우 빠름 |
- **자료구조**: 배열, 리스트, 힙
- **활용 예시**: K번째 수, 가장 큰 수, 정렬된 병합

### 🔸 탐색 (Search)

- **대표 알고리즘**:
    
    
    | 탐색 알고리즘 | 시간 복잡도 | 자료구조 사용 | 주요 특징 |
    | --- | --- | --- | --- |
    | 선형 탐색 (Linear Search) | O(n) | 배열 등 | 앞에서부터 순차 탐색 |
    | 이진 탐색 (Binary Search) | O(log n) | 배열 (정렬 필수) | 중간 값을 기준으로 절반씩 탐색 |
    | DFS (Depth-First Search) | O(V + E) | 스택 / 재귀 | 깊이 우선, 재귀 호출 또는 스택 사용 |
    | BFS (Breadth-First Search) | O(V + E) | 큐 | 너비 우선, 최단 경로 탐색에 활용 |
- **자료구조**: 배열, 그래프, 트리, 스택, 큐
- **활용 예시**: 미로 찾기, 단어 변환, 타겟 넘버

### 🔸 동적 계획법 (Dynamic Programming)

- **정의**: 이전 결과를 저장해 중복 계산 제거
- **전략**: Memoization (Top-down), Tabulation (Bottom-up)
- **자료구조**: 배열, 2D 테이블
- **활용 예시**: 피보나치 수, 계단 오르기, 정수 삼각형, 연속합

### 🔸 그리디 (Greedy)

- **정의**: 매 단계에서 최적의 선택을 함
- **조건**: 항상 전체 최적 해를 보장하지 않음
- **자료구조**: 배열, 힙
- **활용 예시**: 체육복, 조이스틱, 회의실 배정, 구명보트

### 🔸 최단 경로 (Shortest Path)

- **대표 알고리즘**:
    
    
    | 알고리즘 | 시간 복잡도 | 음수 간선 처리 | 주요 특징 | 활용 예시 |
    | --- | --- | --- | --- | --- |
    | 다익스트라 (Dijkstra) | O(E log V) | ❌ 불가능 | 힙(우선순위 큐) 사용, 빠름, 단일 시작점 기준 | 한 정점 → 모든 정점 |
    | 벨만-포드 (Bellman-Ford) | O(VE) | ✅ 가능 | 음수 간선 가능, 느림, 음수 사이클 검출 가능 | 환율, 그래프 검증 |
    | 플로이드-워셜 (Floyd-Warshall) | O(V³) | ✅ 가능 | 모든 정점 쌍 간 거리 계산, DP 기반 | 모든 정점 간 거리 |
- **자료구조**: 그래프, 우선순위 큐, 배열
- **활용 예시**: 도시 간 경로, 특정 지점까지 최소 거리 계산

### 🔸 최소 신장 트리 (Minimum Spanning Tree, MST)

- **정의**: 연결된 그래프에서 모든 정점을 최소 비용으로 연결하는 트리
- **대표 알고리즘**:
    
    
    | 알고리즘 | 방식 | 사용 자료구조 | 시간 복잡도 | 주요 특징 |
    | --- | --- | --- | --- | --- |
    | 크루스칼 (Kruskal) | 간선 중심 | 유니온 파인드 (Disjoint Set) | O(E log E) | 간선을 가중치 기준으로 정렬, 사이클 방지 |
    | 프림 (Prim) | 정점 중심 | 우선순위 큐 (Min Heap) | O(E log V) | 방문한 정점에서 인접한 최소 간선 선택 |
- **활용 예시**: 전력망 최소 연결, 도로 건설 비용 절감
- **대표 문제**: 네트워크 연결 (백준 1922), 도시 분할 계획 (백준 1647)
